Requirements

PHP 5.3.9+ is required for this library. However, we have a stable release and developerment branch for PHP <5.3.8 and PHP 5.2.x as well.

Installation

This library follows the zend PSR-0 standards. A number of autoloaders exist which can autoload this library for that reason, but if you are not using one, you can register the OAuth2\Autoloader:

Php
require_once('/path/to/oauth2-server-php/src/OAuth2/Autoloader.php');
OAuth2\Autoloader::register();

Using Composer? Add the following to composer.json:

Json
{
    "require": {
        "bshaffer/oauth2-server-php": "dev-develop",
        ...
    },
    ...
}

And then run composer.phar install

It is highly recommended you check out the v0.9 tag to ensure your application doesn't break from backwards-compatibility issues, but also this means you will not receive the latest changes.

Learning the OAuth2.0 Standard

If you are new to OAuth2, take a little time first to look at the Oauth2 Demo Application and the source code, and read up on OAuth2 Flows. For everything else, consult the OAuth2.0 Specification

Get Started With oauth2-server-php

Here is an example of a bare-bones OAuth2 Server implementation:

Php
$storage = new OAuth2\Storage\Pdo(array('dsn' => $dsn, 'username' => $username, 'password' => $password));
$server = new OAuth2\Server($storage);
$server->addGrantType(new OAuth2\GrantType\AuthorizationCode($storage)); // or any grant type you like!
$server->handleTokenRequest(OAuth2\Request::createFromGlobals())->send();

This library requires you to define a Storage object, containing instrutions on how to interact with objects in your storage layer such as OAuth Clients and Authorization Codes. Built-in storage classes include PDO, Redis, and Mongo. The interfaces allow (and encourage) the use of your own Storage objects to fit your application's implementation.

Once you've created a storage object, pass it to the server object and define which Grant Types your server is to support. See the list of supported Grant Types.

The final step, once the Server object is set up, is to handle the incoming request. Consult the Server Methods, or see the Step-by-Step walkthrough below to familiarize yourself with the types of requests involved in OAuth2.0 workflows.
